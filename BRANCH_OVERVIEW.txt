====================================================
WHAT THIS BRANCH DOES
====================================================

This branch integrates the standalone device token Flask app and the APNs
notification script into the main backend. Previously these were two separate,
disconnected pieces of code. This branch merges them into the main app so that:

  1. The main Flask backend can receive and store iOS device tokens.
  2. The notification script fetches those tokens from the database instead
     of using a hardcoded value, meaning it sends to ALL registered devices.

CHANGES MADE

1. backend/app/database/database.py
   - Added create_device_tokens_table()
       Creates the device_tokens table on app startup if it doesn't exist.
       Table columns: id, device_token (unique), user_id, created_at.

   - Added upsert_device_token(device_token, user_id)
       Inserts a new device token into the table.
       If the token already exists (e.g. app relaunched), it updates the
       user_id and resets created_at. No duplicates.

   - Added get_all_device_tokens()
       Returns a list of every device token string stored in the table.
       Used by the notification script to know who to notify.

====================================================
FULL WORKFLOW
====================================================

STEP 1 — iOS App Registers Its Device Token
  When the iOS app launches, Apple assigns it a device token (unique ID for
  that device + app). The app sends it to the backend:

    POST /api/uploadDeviceToken
    Body: { "deviceToken": "abc123...", "userId": "P0001" }

  The backend saves it to the device_tokens table. If the token already
  exists it is updated, not duplicated.

STEP 2 — Cron Job Fires Every 15 Minutes
  On the EC2 server a cron job runs send_apns_notification.sh every 15 minutes.
  The shell script activates the Python virtual environment and runs
  sendAPNSNotification2.py.

    Cron entry: */15 * * * * /home/ubuntu/code/scripts/send_apns_notification.sh

STEP 3 — Script Fetches All Tokens from the Database
  sendAPNSNotification2.py connects to the database via DATABASE_URL and
  retrieves every device token that has been registered.

STEP 4 — Silent Push Sent to Every Device
  For each token, a silent background notification is sent to Apple's APNs
  server. Silent means no alert, sound, or badge is shown to the user.
  The payload is:

    { "aps": { "content-available": 1 }, "data": { "action": "refresh" } }

STEP 5 — iOS App Wakes Up and Refreshes
  When the silent push arrives iOS wakes the app in the background. The app
  sees action: refresh and fetches new data from the backend (sensor uploads,
  survey responses, etc.) without any user interaction.

Diagram:

  iOS App --> POST /api/uploadDeviceToken --> DB (device_tokens table)
                                                        |
  EC2 Cron (every 15 min) --> sendAPNSNotification2.py reads DB
                                                        |
                              Apple APNs --> Silent push --> iOS App wakes & refreshes
